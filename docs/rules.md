# URL 生成ルールと制約

カスタムURL を生成する際のルールと制約事項です。
これらは生成システム全体に適用されます。

## カスタム ID の制約

自分で ID を指定する場合、以下の制約があります。

- **最大文字数:** 100文字
- **使用禁止文字:** `/` (スラッシュ) は使用できません。
- **システム予約語:** `api` はシステムで使用するため、ID として指定することはできません。
- **重複禁止:** すでに使用されている ID を登録することはできません。
- **マルチバイト文字:** 日本語や記号・絵文字などのマルチバイト文字も使用可能ですが、URL としてはエンコード（変換）された状態で登録されます。
  - 例: ID `English日本語` → `English%E6%97%A5%E6%9C%AC%E8%AA%9E`
  - URLの使用環境によっては正常に動作しない場合があるため、注意してご利用ください（生成時に警告が表示されます）。

## ランダム ID の生成ルール

ID を指定しない場合、以下の設定に基づいてランダムな ID が生成されます。

- **使用文字種の選択:** 英大文字・英小文字・数字から選択できます。
- **デフォルト設定:**
  - 英小文字: 使用する
  - 数字: 使用する
  - 英大文字: 使用しない
  - 長さ: 6文字
- **制約:** 少なくとも1つ以上の文字種を選択する必要があります。すべてを「使用しない」に設定して ID を生成することはできません。

## 有効期限の設定

リンクの有効期限を指定する場合、以下の形式で指定します。

- **利用可能な単位:** 秒・分・時間・日
- **無期限:** 指定しない場合、リンクは無期限となります。

## その他のルール・制約

- **ベース URL:** 有効な URL 形式（`http://` または `https://` で始まるもの）である必要があります。
- **その他デフォルトの設定値:**
  | 項目 | デフォルト設定 |
  | :-- | :-- |
  | クッションページ | 挟まない (OFF) |
  | 操作ページ | 作成する (ON) |
